## Host Modules

Instead of writing host functions by hand, you can write a Kotlin class containing annotated methods
and let the Kotlin Runtime Web Assembly KSP processor generate the host functions for you. This is especially
useful when you have many host functions.

```kotlin
@HostModule("demo")
class Demo(private val random: Random) {
    @WasmExport
    fun add(a: Int, b: Int): Long = (a + b).toLong()

    @WasmExport // the Wasm name is random_get
    fun randomGet(memory: Memory, ptr: Int, len: Int) {
        val data = ByteArray(len)
        random.nextBytes(data)
        memory.write(ptr, data)
    }

    fun toHostFunctions(): Array<HostFunction> = Demo_ModuleFactory.toHostFunctions(this)
}
```

The `@HostModule` annotation marks the class as a host module and specifies the module name for
all the host functions. The `@WasmExport` annotation marks a method as host function and optionally
specifies the name of the function. If the name is not specified, then the Kotlin function name is
converted from camel case to snake case, as is a common convention in Wasm.

The `Demo_ModuleFactory` object in `toHostFunctions()` is generated by the KSP processor.

Host functions must be instance methods of the class. Static methods are not supported.
This is because host functions will typically interact with instance state in the host class.

To use the host module, instantiate it and fetch the host functions:

```kotlin
import uk.shusek.krwa.runtime.ImportValues

val demo = Demo(Random())
val imports = ImportValues.builder().addFunction(*demo.toHostFunctions()).build()
```

### Type conversions

The following conversions are supported:

| Kotlin Type       | Wasm Type  |
|-------------------|------------|
| `Int`             | `i32`      |
| `Long`            | `i64`      |
| `Float`           | `f32`      |
| `Double`          | `f64`      |

`String` parameters are supported when annotated with `@Buffer` or `@CString`.
`Memory` and `Instance` parameters are injected from the current execution context and do not
become Wasm parameters.

## Module Interfaces

:::tip
Use the [build-time compiler](../execution/build-time-compiler.md) `moduleInterface` parameter for typed module exports and imports. See [Generating Module Exports and Imports](../execution/build-time-compiler.md#generating-module-exports-and-imports).
:::

The KSP processor focuses on Kotlin host modules. For module interface generation, prefer the
build-time compiler path.

## Enabling KSP

Enable KSP and add the annotations plus processor in Gradle:

```kotlin
plugins {
    kotlin("jvm")
    id("com.google.devtools.ksp")
}

dependencies {
    compileOnly("uk.shusek.krwa:annotations:latest-release")
    runtimeOnly("uk.shusek.krwa:annotations:latest-release")
    ksp("uk.shusek.krwa:annotations-processor:latest-release")
}
```

Generated Kotlin sources are written under `build/generated/ksp`.
